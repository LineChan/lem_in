8 # Subject :pushpin:
9 
10 Lem_in is a path finding game. The goal is to find the quickest way to get all ants from an anthill from the entrance to the exit. At the **beginning of the game**, all the ants are in the room ##start. At **each turn**, an ant can only move once through a tube. Be careful, the receiving room must be empy because it can only contain one ant at a time (except for the ##end room). The game is done when all the ants are at the exit.

108 ## Path finding algorithm
109 
110 The anthill with its rooms and tubes can be analyzed as a graph that is:
111 - **unweighted** : all the edges have the same weight (it costs one turn each time an ant moves)
112 - **undirected** : there is no directed edges (an ant go through a tube in both directions)
113 - **cyclic** : there are circles in the graph (possibility to go twice in the same room withe same ant)
114 
115 To find solutions I decided to implement an **Iterative Deepening Depth-First Search** (IDDFS). I chose that algorithm because it combines Breadth-First Search's ([BSF](http://www.geeksforgeeks.org/breadth-first-traversal-for-a-graph/)) fast search (for vertices closer to root) and Depth-Dirst Search's ([DSF](http://www.geeksforgeeks.org/depth-first-traversal-for-a-graph/)) space-efficiency. Also, the IDDSF guarentees that each path found is **the shortest one** available.
116 
117 The IDDSF calls DFS for different depths starting from an initial value. In every call, DFS is restricted from going beyond given depth. Because it is a DSF executed in a BFS fashion, the algorithm is easy to adapt to find **multiples shortest paths** by giving different initial values.
118 
119 ![graph](/img/readme_img.png)
120 
121 Top level nodes are visited **multiples times**. The last (or max depth) level is visited once, second last level is visited twice, and so on. It may seem expensive, but it turns out to be not so costly, since in a tree most of the nodes are in the bottom level. So it does not matter much if the upper levels are visited multiple times.
122 
123 Because the IDDFS works only on **acyclic** graphs, I used an [adjacency matrix](https://en.wikipedia.org/wiki/Adjacency_matrix) I adapted so a vertex can't be visited multiple times (see :arrow_down: DLS). The matrix can take 3 values :
124 - 1 : if the pairs of vertices are adjacent (there is a tube between two rooms)
125 - 0 : if not
126 - -1 : if the vertex has already been visited
127 
128 ### IDDFS
129 In our case, the **maximum depth** is the number of rooms (ROOM_NB). Indeed, if there is a solution, the longest path possible goes through all rooms once. Before starting the IDDFS, the **maximal number of shortest paths** (SP_NB) is also defined. We simply look the number of rooms linked to the ##start and ##end rooms and take the lower one.
130 
131 Example : the ##start rooms has 2 tubes and the ##end room has 3 tubes--> SP_NB = 2 . In the best case, both paths are shortest paths from ##start to **2 out of 3 adjacent rooms** of ##end.
132 
133 ```C
134 int ft_iddfs(void)
135 {
136  int limit;
137 
138  limit = 0;
139 
140  /* Loop until all the shortest path aren't found */
141  while (SP_NB > 0)
142  {
143  /* Repeadted Depth-Limit Search until the maximum depth */
144  while (limit < ROOM_NB)
145  {
146  if (ft_dls(START_REF, limit))
147  break ;
148  ++limit;
149  }
150  --SP_NB;
151  }
152  if (ft_lst_is_head(&SHORTEST_PATH))
153  return (0);
154  return (1);
155 }
156 ```
157 
158 ### DLS
159 Depth-Limit Search is a recursive function that starts from a given source and stops when the ##end room is found or the depth limit is reached (no solution).
160 
161 
162 ```C
163 static int ft_dls(const int src_ref, const int limit)
164 {
165  int i;
166 
167  /* Result found */
168  if (src_ref == END_REF)
169  return (1);
170 
171  /* Limit depth reached*/
172  if (limit <= 0)
173  {
174  ft_reset_matrix();
175  return (0);
176  }
177  i = 0;
178 
179  /* Set the node as visited */
180  while (i < ROOM_NB)
181  {
182  if (!(MATRIX[i][src_ref] ^ 1))
183  MATRIX[i][src_ref] = -1;
184  ++i;
185  }
186 
187  /* Start DLS from adjacent nodes */
188  if (ft_explore_adjacent(src_ref, limit))
189  return (1);
190  return (0);
191 }
192 ```
193 
194 ### Exploring adjacent nodes
195 
196 ```C
197 static int ft_explore_adjacent(const int ref, const int limit)
198 {
199  int i;
200  int j;
201 
202  i = 0;
203  while (i < ROOM_NB)
204  {
205  /* If it is an adjancent and on the shortest path */
206  if ((MATRIX[ref][i] == 1) && (ft_dls(i, limit - 1)))
207  {
208  if (i != END_REF)
209  {
210  /* If it isn't the ##end room, move the room from the anthill
211  to the shortest path list */
212  ft_lst_moveto_next(ft_find_room_with_ref(&ANTHILL, i), &PATH(SHORTEST_PATH.prev));
213  j = 0;
214  /* Removing the room from matrix by setting the value to 0 */
215  while (j < ROOM_NB)
216  ft_set_matrix(i, j++, 0);
217  }
218  else
219  {
220  /* Create a shortest path list and copy the ##end room at the end of it */
221  /* NB : Because the IDDFS always finds the shortest path,
222  solutions are sorted from the shortest to the longest) */
223  ft_add_shortest_path();
224  ft_copy_end_room_to_path();
225  }
226  ++PATH_LEN(SHORTEST_PATH.prev);
227  return (1);
228  }
229  ++i;
230  }
231  return (0);
232 }
233 ```

235 ## Computing moves
236 
237 Now that we have SP_NB shortest path, ants need to be sent on these paths. To minimize the number of moves, paths are taken according to these rules :
238 - **1st shortest path** (best solution) : an ant is sent at each turn
239 - **Other paths** : an ant is sent if only it is less cost-effective to take the current path than the shortest one.
240 
241 ```C
242 if (PATH_LEN(current_path) <= ((ANT_NB - current_ant) * PATH_LEN(BEST_SHORTEST_PATH)))
243  ft_sent_ant(&PATH(current_path), &current_ant);
244 ```
